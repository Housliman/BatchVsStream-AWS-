# 2. نظرة معمقة: المعالجة بالدُفعات


## الخصائص:

- [ ] **حجم البيانات:** مجموعات بيانات كبيرة ومحدودة.
- [ ] **زمن الاستجابة:** زمن استجابة عالٍ (من دقائق إلى ساعات).
- [ ] **الإنتاجية:** الإنتاجية العالية هي الهدف الأساسي.
- [ ] **التنفيذ:** وظائف مجدولة ودورية.

## حالات الاستخدام الشائعة:

- [ ] تقارير وتحليلات نهاية اليوم.
- [ ] أنظمة الرواتب والفوترة.
- [ ] وظائف ETL (استخراج، تحويل، تحميل) واسعة النطاق.
- [ ] تحليل الجينوم.

## خدمات Google Cloud الرئيسية للمعالجة بالدُفعات:

- [ ] **Google Cloud Storage:** تخزين بحيرة البيانات للبيانات الخام والمعالجة.
- [ ] **Cloud Data Fusion & Dataproc:** خدمة ETL بدون خادم و Spark/Hadoop مُدارة لإعداد البيانات وتحميلها.
- [ ] **Dataproc:** منصة بيانات ضخمة مُدارة (Spark, Hadoop, Hive).
- [ ] **Google Cloud Batch:** حوسبة دُفعات مُدارة بالكامل لجدولة وتنفيذ الوظائف.
- [ ] **BigQuery:** مستودع بيانات للتحليلات واسعة النطاق.

## الإيجابيات والسلبيات:

- [ ] **الإيجابيات:** فعالة من حيث التكلفة لمجموعات البيانات الكبيرة، إنتاجية عالية، بنية بسيطة.
- [ ] **السلبيات:** زمن استجابة عالٍ، الرؤى ليست في الوقت الفعلي، يمكن أن تكون إدارتها معقدة للاستعلامات المخصصة.
