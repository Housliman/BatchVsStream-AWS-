# 3. نظرة معمقة: المعالجة بالبث


## الخصائص:

- [ ] **حجم البيانات:** تدفقات بيانات صغيرة، غير محدودة، ومستمرة.
- [ ] **زمن الاستجابة:** زمن استجابة منخفض (من أجزاء من الثانية إلى ثوانٍ).
- [ ] **الإنتاجية:** يمكنها التعامل مع بيانات عالية السرعة.
- [ ] **التنفيذ:** معالجة مستمرة قائمة على الأحداث.

## حالات الاستخدام الشائعة:

- [ ] كشف الاحتيال في الوقت الفعلي في المعاملات المالية.
- [ ] مراقبة بيانات مستشعرات إنترنت الأشياء والتنبيه.
- [ ] لوحات الصدارة المباشرة لتطبيقات الألعاب.
- [ ] تحليل مسار النقرات (Clickstream) للمواقع الإلكترونية وتطبيقات الجوال.

## خدمات Google Cloud الرئيسية للمعالجة بالبث:

- [ ] **Pub/Sub:** لاستيعاب وتخزين تدفقات البيانات في الوقت الفعلي.
- [ ] **Dataflow:** لتحميل وتحويل ومعالجة بيانات البث.
- [ ] **Google Cloud Functions:** حوسبة بدون خادم قائمة على الأحداث لمعالجة الأحداث الفردية.
- [ ] **خدمات Kafka المُدارة (مثل Confluent Cloud):** لسير عمل Kafka المُدار.
- [ ] **Dataproc:** لتشغيل Spark Streaming أو Flink على مجموعة مُدارة.

## الإيجابيات والسلبيات:

  - [ ] **الإيجابيات:** رؤى في الوقت الفعلي، استجابة فورية للأحداث، قابلة للتطوير بشكل كبير.
  - [ ] **السلبيات:** يمكن أن تكون أكثر تعقيدًا في التصميم والإدارة، تكاليف تشغيلية أعلى محتملة، تتطلب معالجة البيانات غير المرتبة.
